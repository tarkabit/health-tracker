## [2026-06-24] — Session 1
**Goal:** Build a fast, clean native macOS habit tracker that stores each habit's record as
Markdown, then import some existing history into it.

**Built:**
- Full SwiftUI app as a Swift Package executable: flexible habit model (fields / targets /
  cadence / goal), `VaultStore` + `HabitFile` (Yams) + `LogFile` (Markdown table) persistence,
  `AppState` writing every edit to disk immediately.
- Views: Today screen with inline one-gesture loggers, habit detail (heatmap, weekly
  progress, editable history), New/Edit habit form. Six seeded habits.
- `build.sh` assembles `Health Tracker.app`; custom app icon generated by
  `Tools/makeicon.swift` (CoreGraphics) → `AppIcon.icns`.
- One-off import: parsed existing history from local Markdown files into a couple of
  habits (date + per-field values, with run sessions classified into the habit's choices).
- Heatmap upgrade (all habits): month + weekday labels, "today" outline, clickable days
  (tap to log/edit), and a goal-aware headline stats row.

**Decisions made:**
- SwiftUI native over Electron/Tauri (fastest launch, cleanest, Xcode present).
- Markdown files are the single source of truth — no database.
- Vault stored inside the iCloud project folder (`HabitData/`).
- Running `distance` modeled as a `note` field (counters are integer-only; distances are decimal).
- Manifest pinned to Swift 5 language mode (tools 5.9).

**Deferred:**
- No automated test target.
- Counter-driven heatmap shading (offered, declined).
- Some source data existed only as images; not OCR'd (the text log was sufficient).

**Issues:** None outstanding. Background GUI-launch verification was declined mid-session;
verified instead via seeding to a temp vault + structural lint of the generated logs + clean builds.

**Next session:** Use the app for daily logging; consider streak stats, pull-up trend chart,
or a menu-bar quick-logger if desired.

**Tests:** No test suite. `swift build` clean; `swift test` → no tests found (expected).
